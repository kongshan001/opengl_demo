# REQ-011: 提升单元测试覆盖率到 80%

**优先级**: P0
**类型**: 优化
**预估工时**: 2 小时
**状态**: 开发中

## 用户故事

作为一个开发者，
我希望单元测试覆盖率达到 80%，
以便确保代码质量和稳定性。

## 背景

当前测试覆盖率较低，需要补充测试用例：
- CShader: 需要测试 uniform 设置、错误处理
- CMesh: 需要测试顶点属性、边界情况
- Camera: 需要测试移动、旋转、矩阵计算
- CTexture: 需要测试加载、边界情况

## 验收标准

- [ ] CShader 测试覆盖率 >= 80%
- [ ] CMesh 测试覆盖率 >= 80%
- [ ] Camera 测试覆盖率 >= 80%
- [ ] CTexture 测试覆盖率 >= 80%
- [ ] 总体覆盖率 >= 80%

## 技术要点

- 使用 Google Test 框架
- 边界条件测试（空输入、无效输入）
- 异常测试（使用 EXPECT_THROW）
- Mock OpenGL 调用（可选）

## 测试要求

- 每个公共方法至少一个测试
- 边界条件测试
- 异常情况测试
